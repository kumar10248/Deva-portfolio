Tailwind CSS has revolutionized how we write styles in modern web development. Its utility-first approach makes styling faster and more maintainable. But there's so much more to Tailwind than just adding classes!

In this article, I'll share some powerful tips and tricks that will help you become more productive with Tailwind CSS.

## 1. Use @apply for Repeated Patterns

When you find yourself using the same combination of utilities repeatedly, extract them using `@apply` in your CSS.

**Instead of:**

<CodeWindow title="Repeated Classes">
```html
<button class="px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600">
  Button 1
</button>
<button class="px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600">
  Button 2
</button>
```
</CodeWindow>

**Do this:**

<CodeWindow title="styles.css">
```css
/* In your CSS file */
.btn-primary {
  @apply px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600;
}
```
</CodeWindow>

<CodeWindow title="Component">
```html
<button class="btn-primary">Button 1</button>
<button class="btn-primary">Button 2</button>
```
</CodeWindow>

## 2. Customize Your Colors in tailwind.config.js

Don't limit yourself to Tailwind's default colors. Add your brand colors to make them available as utilities.

<CodeWindow title="tailwind.config.js">
```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        'brand-blue': '#0066CC',
        'brand-purple': '#6366F1',
        'success': '#10B981',
        'warning': '#F59E0B',
      }
    }
  }
}
```
</CodeWindow>

## 3. Use Arbitrary Values for One-Off Designs

Sometimes you need a specific value that doesn't exist in Tailwind. Use arbitrary values with square brackets:

<CodeWindow title="Arbitrary Values">
```html
<div class="top-[117px]">Positioned exactly 117px from top</div>
<div class="bg-[#1da1f2]">Twitter blue background</div>
<div class="grid-cols-[200px_1fr_1fr]">Custom grid columns</div>
```
</CodeWindow>

## 4. Master the Group Modifier

The `group` modifier lets you style children based on parent hover or other states:

<CodeWindow title="Group Hover">
```html
<div class="group">
  <img src="photo.jpg" class="group-hover:scale-110 transition" />
  <h3 class="group-hover:text-blue-500">Hover the parent!</h3>
</div>
```
</CodeWindow>

When you hover over the parent div, both the image and heading change!

## 5. Use Peer for Sibling Interactions

Style an element based on its sibling's state using the `peer` modifier:

<CodeWindow title="Peer Modifier">
```html
<input type="checkbox" class="peer" />
<label class="peer-checked:text-blue-500 peer-checked:font-bold">
  I change when checkbox is checked!
</label>
```
</CodeWindow>

This is perfect for custom form controls!

## 6. Create Custom Utilities

Add your own utilities to the Tailwind config:

<CodeWindow title="tailwind.config.js">
```javascript

  theme: {
    extend: {
      spacing: {
        '128': '32rem',
        '144': '36rem',

      borderRadius: {
        '4xl': '2rem',

    }
  }
```

</CodeWindow>

## 7. Use Container Queries (New!)

Tailwind v3.2+ supports container queries. Style elements based on container size, not viewport size:

<CodeWindow title="Container Queries">
```html
<div class="@container">
  <div class="@lg:grid-cols-2 grid">
    {/* These columns change based on container width, not screen width */}
  </div>
</div>
```
</CodeWindow>

## 9. Use Dark Mode Effectively

Tailwind makes dark mode super easy with the `dark:` prefix:

<CodeWindow title="Dark Mode">
```html
<div class="bg-white dark:bg-gray-800 text-gray-900 dark:text-gray-100">
  This adapts to dark mode automatically!
</div>
```
</CodeWindow>

Enable it in your config:

<CodeWindow title="tailwind.config.js">
```javascript
module.exports = {
  darkMode: 'class', // or 'media'
}
```
</CodeWindow>

## 12. Create Custom Animations

Add your own animations to the config:

<CodeWindow title="tailwind.config.js">
```javascript
module.exports = {
  theme: {
    extend: {
      keyframes: {
        wiggle: {
          '0%, 100%': { transform: 'rotate(-3deg)' },
          '50%': { transform: 'rotate(3deg)' },
        }
      },
      animation: {
        wiggle: 'wiggle 1s ease-in-out infinite',
      }
    }
  }
}
```
</CodeWindow>

Now use: `animate-wiggle`

## 19. Create Your Own Plugin

Extend Tailwind with custom plugins:

<CodeWindow title="tailwind.config.js">
```javascript
// tailwind.config.js
const plugin = require('tailwindcss/plugin')

module.exports = {
  plugins: [
    plugin(function({ addUtilities }) {
      addUtilities({
        '.text-shadow': {
          textShadow: '2px 2px 4px rgba(0, 0, 0, 0.5)',
        },
        '.text-shadow-lg': {
          textShadow: '4px 4px 8px rgba(0, 0, 0, 0.5)',
        },
      })
    })
  ]
}
```
</CodeWindow>

## 20. Performance Tips

### Purge Unused Styles

Make sure you're purging unused styles in production:

<CodeWindow title="tailwind.config.js">
```javascript
module.exports = {
  content: [
    './src/**/*.{js,jsx,ts,tsx}',
    './public/index.html',
  ],
  // This removes unused styles in production
}
```
</CodeWindow>

### Use JIT Mode

Tailwind's JIT (Just-In-Time) mode generates styles on-demand, making development faster and the final bundle smaller. It's enabled by default in v3+.
